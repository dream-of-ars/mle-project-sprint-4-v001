# Подготовка виртуальной машины

## Склонируйте репозиторий

Склонируйте репозиторий проекта:

```
git clone https://github.com/yandex-praktikum/mle-project-sprint-4-v001.git
```

## Активируйте виртуальное окружение

Используйте то же самое виртуальное окружение, что и созданное для работы с уроками. Если его не существует, то его следует создать.

Создать новое виртуальное окружение можно командой:

```
python3 -m venv env_recsys_start
```

После его инициализации следующей командой

```
. env_recsys_start/bin/activate
```

установите в него необходимые Python-пакеты следующей командой

```
pip install -r requirements.txt
```

### Скачайте файлы с данными

Для начала работы понадобится три файла с данными:
- [tracks.parquet](https://storage.yandexcloud.net/mle-data/ym/tracks.parquet)
- [catalog_names.parquet](https://storage.yandexcloud.net/mle-data/ym/catalog_names.parquet)
- [interactions.parquet](https://storage.yandexcloud.net/mle-data/ym/interactions.parquet)
 
Скачайте их в директорию локального репозитория. Для удобства вы можете воспользоваться командой wget:

```
wget https://storage.yandexcloud.net/mle-data/ym/tracks.parquet

wget https://storage.yandexcloud.net/mle-data/ym/catalog_names.parquet

wget https://storage.yandexcloud.net/mle-data/ym/interactions.parquet
```

```
jupyter lab --ip=0.0.0.0 --no-browser
```

# Расчёт рекомендаций

Код для выполнения первой части проекта находится в файле `recommendations.ipynb`. Изначально, это шаблон. Используйте его для выполнения первой части проекта.

# Сервис рекомендаций

Код сервиса рекомендаций находится в файле `recommendations_service.py`.

## Как запустить сервис

1. Активируйте виртуальное окружение:

```
. env_recsys_start/bin/activate
```

2. Убедитесь, что в корне репозитория есть файл `.env` с доступом к S3.

3. Запустите сервис:

```
uvicorn recommendations_service:app --host 0.0.0.0 --port 8000
```

4. Проверьте состояние сервиса:

```
curl http://127.0.0.1:8000/health
```

Сервис сначала ищет файлы рекомендаций локально, затем пробует скачать их из S3 по пути `recsys/recommendations/`. Если артефакты недоступны, сервис строит рабочий fallback из `events.parquet`.

## Стратегия смешивания онлайн- и офлайн-рекомендаций

- Офлайн-база: `recommendations.parquet`.
- Резервный офлайн-слой: `personal_als.parquet`.
- Глобальный fallback: `top_popular.parquet`.
- Онлайн-сигнал: история пользователя после `2022-12-16` из `events.parquet` + i2i-кандидаты из `similar.parquet`.

Итоговая выдача формируется смешиванием офлайна и онлайна в пропорции около `70/30` с дедупликацией и фильтрацией треков из онлайн-истории пользователя.
Если персональных офлайн-рекомендаций нет, сервис возвращает `top_popular`, при наличии онлайн-истории добавляет i2i-кандидаты.

# Инструкции для тестирования сервиса

Код для тестирования сервиса находится в файле `test_service.py`.

## Как запустить тестовый скрипт

1. Активируйте виртуальное окружение:

```
. env_recsys_start/bin/activate
```

2. Запустите тесты и сохраните вывод:

```
python test_service.py > test_service.log 2>&1
```

Скрипт автоматически:
- поднимает сервис через `uvicorn`,
- проверяет три обязательных сценария:
  - пользователь без персональных рекомендаций,
  - пользователь с персональными рекомендациями без онлайн-истории,
  - пользователь с персональными рекомендациями и онлайн-историей,
- завершает сервис после проверок.
